Workflow / 工作流

A growth system you can inspect

一个可检查、可修改、可审查的增长系统，而不是自动 spam 机器人。

Listen

Find conversations where your audience already shows pain, intent, or confusion.

发现用户已经表达痛点、意图或困惑的真实对话。

Plan

Convert channel context into small tasks your agent can prepare and you can review.

把渠道上下文转成 Agent 可准备、你可审核的小任务。

Draft

Create specific posts and replies that help first and promote only when appropriate.

生成具体、有帮助、不过度推广的帖子和回复草稿。

Remember

Keep lightweight notes so the next plan improves instead of starting over.

保留轻量记录，让下一轮计划变好，而不是每天从零开始。

Safety / 安全边界

Built for human-in-the-loop growth

Lotus Growth is not an auto-spam framework. The default guardrails require approval before public actions, keep runtime data private, and avoid platform bypass behavior.

Lotus Growth 不是自动发帖框架。默认边界是公开动作前人工确认、私有运行数据不入库、不绕过平台规则。

  • No public post without review / 不经审核不公开发布
  • No committed cookies or session data / 不提交 cookie 或 session
  • No fake personas / 不伪造人设
  • No link-first posting / 不做链接优先的 spam

For Builders / 给开发者

Small enough to understand, useful enough to adapt.

Start with editable JSON config, run a simple CLI loop, and replace the planner with your own growth logic when you have real pressure.

从可编辑 JSON 配置开始，先跑通简单 CLI 闭环。等你有真实压力后，再替换规划逻辑，而不是一开始就搭重平台。
